# 개발일지 - 2026-07-17

## 주제

WeldVision 2단계 개발 시작: YOLOv8 연결 전 단계의 Gradio 기반 해석 대시보드 구축

## 오늘의 목표

1단계에서 만들었던 OpenCV 기반 특징 분석 흐름을 버리지 않고, 2단계 시스템의 해석 두뇌로 재활용할 수 있는 구조를 만든다.

이번 단계에서는 아직 YOLOv8 학습 모델을 붙이지 않고, 먼저 다음 기능을 구현했다.

- 이미지 업로드 UI
- OpenCV 전처리 결과 시각화
- 결함 후보 영역 표시
- 형태 특징값 추출
- 위험도 점수화
- 원인 추론 및 권장 조치 출력

## 구현 내용

### 1. Gradio UI 구성

Streamlit 대신 Gradio를 사용해 데모 화면을 구성했다.

화면에는 다음 영역을 배치했다.

- 용접 이미지 업로드
- YOLOv8 모델 경로 입력
- 원본 이미지
- 검출 결과 이미지
- 전처리 근거 화면
- 슬라이더 조절 영역
- 분석 결과 요약
- 특징값 표

UI 문구는 한글 중심으로 작성하되, 기술 용어는 영어를 병기했다.

예시:

- 국소 대비 강도 (CLAHE)
- 어두운 결함 강조 (Black-hat)
- 방향성/경계 강조 (Gradient)
- 엠보싱 깊이 (Emboss)
- 원형도 (Circularity)
- 종횡비 (Aspect Ratio)
- 평균 밝기 (Mean Brightness)

## 전처리 설계

이번 단계의 핵심 전처리는 다음 4가지로 정리했다.

| 전처리 | 역할 |
|---|---|
| CLAHE | 국소 대비를 강화해 흐릿한 결함을 보기 쉽게 함 |
| Black-hat | 밝은 배경 위의 어두운 결함 후보를 강조 |
| Gradient | 선형 결함, 방향성, 경계 변화를 강조 |
| Emboss | 기공과 슬래그 혼입처럼 질감이 중요한 패턴을 보기 쉽게 함 |

Canny는 핵심 검출 방식에서 제외했다.

이유는 용접부 이미지에서 Canny가 결함뿐 아니라 비드 질감, 노이즈, 스크래치까지 경계로 잡아 과검출될 가능성이 크기 때문이다.

## OpenCV 후보 검출

아직 YOLOv8 모델이 연결되지 않은 상태이므로, 현재 박스는 최종 AI 검출 결과가 아니다.

현재는 Black-hat 전처리로 강조된 어두운 영역을 기반으로 결함 후보를 표시한다.

따라서 결과 요약에 다음 안내를 추가했다.

> YOLOv8 모델이 연결되지 않아 OpenCV 전처리 기반 후보 모드로 표시 중입니다. 이 박스는 최종 AI 판정이 아니라, Black-hat에서 강조된 어두운 영역 후보입니다.

작은 노이즈가 결함처럼 과검출되는 문제를 줄이기 위해 다음 조건을 보수적으로 조정했다.

- 최소 후보 면적 증가
- 작은 폭/높이 영역 제외
- contour extent 기준 추가
- 후보 개수 제한
- 기본 어두운 후보 임계값 상향

## 특징값 추출

1단계에서 사용했던 특징값을 2단계에서도 그대로 재사용할 수 있도록 구성했다.

| 특징값 | 활용 |
|---|---|
| 원형도 (Circularity) | 기공처럼 둥근 결함 후보 판단 |
| 종횡비 (Aspect Ratio) | 균열, 슬래그 혼입, 용입불량처럼 길쭉한 결함 판단 |
| 평균 밝기 (Mean Brightness) | 어두운 결함 후보의 농도 확인 |
| 면적 비율 (Area Ratio) | 결함 크기와 영향 범위 확인 |

## 위험도 및 원인 추론

결함 종류별로 위험도 점수, 추정 원인, 권장 조치를 룰 기반으로 정의했다.

| 결함 | 위험도 | 권장 조치 |
|---|---:|---|
| 균열 (Crack) | 100 | 즉시 재작업 |
| 용입불량 (Lack of Fusion) | 80 | 재검사 또는 보수 용접 |
| 언더컷 (Undercut) | 60 | 보수 용접 |
| 기공 (Porosity) | 50 | 주의 관찰, 밀집 시 재검사 |
| 슬래그 혼입 (Slag Inclusion) | 40 | 크기와 위치에 따라 추가 확인 또는 경미 보수 |

## 실시간 슬라이더 반영

버튼을 누르는 방식뿐 아니라, 이미지가 업로드된 상태에서 슬라이더 값이 바뀌면 전처리 화면과 후보 검출 결과가 자동 갱신되도록 연결했다.

단, 이것은 영상 프레임 단위의 실시간 검출이 아니라 Gradio 이벤트 기반 자동 재분석이다.
