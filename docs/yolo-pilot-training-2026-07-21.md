# YOLOv8 파일럿 학습 결과 (2026-07-21)

## 목적

RIAWELC 폴리곤 JSON을 YOLO 바운딩 박스로 변환한 뒤, 데이터 변환부터 학습·검증·Gradio 추론까지 전체 경로가 실제 규모의 표본에서도 동작하는지 확인했다. 이번 결과는 최종 모델 성능이 아니라 본 학습 규모와 개선 방향을 결정하기 위한 파일럿 실험이다.

## 데이터 구성

원본 Training/Validation의 15개 폴더에서 폴더별 최대 100장을 고정 시드로 추출했다. 정상 이미지는 결함 클래스가 아닌 빈 라벨의 음성 샘플로 포함했다.

| 구분 | 이미지 | 음성 이미지 | 결함 객체 |
|---|---:|---:|---:|
| Train | 1,500 | 300 | 4,329 |
| Validation | 1,368 | 300 | 4,572 |

결함 객체 수는 한 이미지에 여러 기공이 표시되는 데이터 특성 때문에 이미지 수보다 많다. 좌표가 비어 있거나 면적이 0인 잘못된 객체는 Train 3개, Validation 4개를 건너뛰었다.

## 학습 설정

| 항목 | 값 |
|---|---|
| 모델 | YOLOv8n pretrained |
| Epochs | 10 |
| Image size | 512 |
| Batch size | 8 |
| Device | Apple M4 Pro MPS |
| Optimizer | AdamW (Ultralytics auto) |
| 소요 시간 | 약 8분 43초 |

실행 명령:

```bash
python phase2/train_yolo.py \
  --data phase2/yolo_dataset_pilot/dataset.yaml \
  --epochs 10 \
  --imgsz 512 \
  --batch 8 \
  --workers 0 \
  --name pilot-100x10
```

## 전체 검증 결과

| Precision | Recall | mAP50 | mAP50-95 |
|---:|---:|---:|---:|
| 0.421 | 0.390 | 0.358 | 0.156 |

mAP50은 1 epoch의 0.056에서 10 epoch의 0.358로 지속 상승했다. 마지막 epoch가 가장 높은 검증 성능을 기록해, 현재 표본에서는 아직 충분히 수렴하지 않은 것으로 보인다.

## 클래스별 결과

| Class | Validation images | Instances | Precision | Recall | mAP50 | mAP50-95 |
|---|---:|---:|---:|---:|---:|---:|
| crack | 139 | 264 | 0.329 | 0.333 | 0.277 | 0.103 |
| porosity | 301 | 3,209 | 0.624 | 0.482 | 0.491 | 0.188 |
| lack_of_fusion | 233 | 332 | 0.380 | 0.170 | 0.192 | 0.083 |
| slag_inclusion | 202 | 409 | 0.337 | 0.186 | 0.188 | 0.065 |
| incomplete_penetration | 100 | 140 | 0.419 | 0.763 | 0.629 | 0.312 |
| undercut | 100 | 218 | 0.436 | 0.408 | 0.372 | 0.185 |

용입부족과 기공은 상대적으로 빠르게 학습됐고, 융합불량과 슬래그혼입은 Recall이 낮아 더 많은 표본과 학습이 필요하다. 기공은 객체 수가 다른 클래스보다 크게 많으므로 다음 실험에서는 이미지 수뿐 아니라 객체 수 불균형도 함께 확인해야 한다.

## Gradio 추론 연결 확인

생성된 `best.pt`를 기존 `detect_with_yolo` 함수에 연결해 기본 신뢰도 0.25에서 추론했다.

- `VT_ST_03_14608597.jpg`: incomplete_penetration 0.776, 0.494
- `VT_ST_06_14627792.jpg`: undercut 0.309
- `VT_ST_02_14600480.jpg`: porosity 최고 0.606

일부 이미지에서는 아직 검출 결과가 없었으므로 파일럿 모델을 최종 판정용으로 사용하면 안 된다. 모델 파일과 생성된 학습 결과는 Git에 포함하지 않는다.

## 다음 실험

1. 폴더별 표본을 500~1,000장으로 확대한다.
2. 30~50 epoch 학습과 early stopping을 적용한다.
3. 융합불량·슬래그혼입의 Recall을 우선 개선한다.
4. 객체 수 및 결함 크기 분포를 분석해 클래스/크기 불균형을 점검한다.
5. 최종 모델은 별도 테스트셋과 실제 업로드 이미지로 확인한 뒤 Gradio 기본 모델로 연결한다.
